# 支持与问题分流 · Support

## 中文

### 使用问题与反馈

请使用本仓库的 GitHub Issues。提交前先搜索已有问题；如果是使用问题，请在标题前加 `[Question]`，并说明：

- Ruyi 版本、Windows 版本和 Node.js 版本；
- 使用的引擎类型（OpenAI 兼容端点或 Claude CLI，不要提供密钥）；
- 可安全公开的复现步骤、预期结果与实际结果；
- 已脱敏的错误信息、日志片段或截图。

功能建议请说明要解决的工作流、目标用户和可接受的替代方案。Bug 与功能请求优先使用仓库提供的 Issue 表单。

### 安全问题

**不要在公开 Issue、PR 或讨论中报告未修复的漏洞，也不要粘贴 API 密钥、访问 token 或个人数据。** 请通过 GitHub Security Advisories 私密报告，并遵循 [`SECURITY.md`](./SECURITY.md)。

### 贡献支持

准备提交 Pull Request 前，请阅读 [`CONTRIBUTING.md`](./CONTRIBUTING.md)。其中定义了本项目必须保持的离线可用、clean-room、Windows 兼容及 e2e 回归约束。

## English

### Usage questions and feedback

Use this repository's GitHub Issues. Search existing issues first; for a usage question, prefix the title with `[Question]` and include:

- the Ruyi, Windows, and Node.js versions;
- the engine type (OpenAI-compatible endpoint or Claude CLI—never the key itself);
- safely shareable reproduction steps, expected behavior, and actual behavior; and
- sanitized errors, log excerpts, or screenshots.

For a feature proposal, explain the workflow, target users, and acceptable alternatives. Prefer the repository's Issue forms for bugs and feature requests.

### Security issues

**Do not report an unfixed vulnerability in a public Issue, Pull Request, or discussion, and never paste API keys, access tokens, or personal data.** Report it privately through GitHub Security Advisories and follow [`SECURITY.md`](./SECURITY.md).

### Contribution support

Read [`CONTRIBUTING.md`](./CONTRIBUTING.md) before opening a Pull Request. It defines the project's required offline-first, clean-room, Windows-compatibility, and e2e-regression constraints.
