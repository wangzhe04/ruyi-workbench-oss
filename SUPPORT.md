# 支持与问题分流 · Support

## 使用问题与反馈 · Usage questions and feedback

请使用本仓库的 GitHub Issues。提交前先搜索已有问题；如果是使用问题，请在标题前加 `[Question]`，并说明：

- Ruyi 版本、Windows 版本和 Node.js 版本；
- 使用的引擎类型（OpenAI 兼容端点或 Claude CLI，不要提供密钥）；
- 可安全公开的复现步骤、预期结果与实际结果；
- 已脱敏的错误信息、日志片段或截图。

功能建议请说明要解决的工作流、目标用户和可接受的替代方案。Bug 与功能请求优先使用仓库提供的 Issue 表单。

## 安全问题 · Security issues

**不要在公开 Issue、PR 或讨论中报告未修复的漏洞，也不要粘贴 API 密钥、访问 token 或个人数据。** 请通过 GitHub Security Advisories 私密报告，并遵循 [`SECURITY.md`](./SECURITY.md)。

## 贡献支持 · Contribution support

准备提交 Pull Request 前，请阅读 [`CONTRIBUTING.md`](./CONTRIBUTING.md)。其中定义了本项目必须保持的离线可用、clean-room、Windows 兼容及 e2e 回归约束。
