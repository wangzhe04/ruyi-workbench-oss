状态:设计稿(2026-07-10,基于 v1.5.0 `a474557`)。作者:主会话亲自通读 index.html(716 行)/ styles.css(1563 行,全文)/ app.js 结构 + 1366/375 双端实机走查后撰写。
定位:v1 视觉系统(青花,`UI-ORCHESTRATION-REDESIGN.md` §3)解决了"有没有";本稿解决"落没落地、精不精"。

## 0. 审计总结论(全部有实测证据)

**这套前端的骨架是对的**——三栏布局、popover 原语、语义色 token、分级 UI、XSS 纪律(el()/textContent)、focus-visible/reduced-motion 无障碍基线,都在。真正的问题集中在四层:

| # | 结论 | 证据 |
|---|------|------|
| A | **设计系统"纸面完成"**:§3.1 补的 token 大面积没接线 | 24 个死令牌(`--elev-2/3`、`--dur-fast/base/slow`、`--ease-out`、全套语义 `-bg/-fg`、`--fs-base/lg`、`--sp-1/7`、`--density-scale`、`--gold-soft`、`--link`);2 个引用未定义(`--fg`、`--text`×6 处);审计徽标硬编码 `#2f6fb0`/`#c9772f`;工作流区散落 `var(--danger,#d9534f)` 式防御字面量 |
| B | **简单模式的"舒适化"实际失效** | html 根字号 15px 但 body 硬编码 14px,全站 **290 处字号全 px、0 rem** → +1px 意图被吞;`--density-scale` 零使用;`--tap-min` 只挂了 3 个选择器(starter-chip/empty-cta/boot 按钮),composer 按钮、会话项、页签全不受密度联动 |
| C | **v1.5 的信息重心与布局错位**:最重的功能(工作流监控/任务池审批/用量看板)全塞在 340px 右栏 | 右栏内容宽 312px;agent-run 卡=聚合头+节点卡+池审批卡+插话按钮层层嵌套;用量看板的条形图/趋势图在 312px 里施展不开 |
| D | **移动端顶栏真断裂** | 375px 实测:chips 换行两行,topbar 定高 54px,第一行 `y=-6` 被裁出视口(工作台/安全/模型 chip 半不可见);≤760px 侧栏 absolute 浮层开机默认展开、盖住对话区 77% 宽 |

另有两条产品级(非像素级)发现:
- **内容语言断层**:内置 18 个技能的名称是英文 mono id(`api-debugger`)、描述整段英文——"中文第一 + 非程序员主画像"的产品把英文说明怼给主画像用户(技能库实测截取:"Use this skill to inspect local or intranet HTTP services…")。
- **图标语言 = emoji 大杂烩**(📁🛡🧰📎⌘🗜🩺⚙☰⋯):跨机渲染不一致、隐喻风格混杂,与精心绘制的如意品牌 SVG 形成强烈反差——这是"看起来不够专业"的最大单一因素。

## 1. 设计系统 v3:从"定义了"到"生效了"

### 1.1 字号体系:px → rem 一刀切(P0,机械改造)
全站 `font-size` 从 px 换算成 rem(base 14px=0.875rem 或直接把 html 设为 14px、body 继承),**删除 body 的硬编码 14px**。收益立现:
- 简单模式 `html{font-size:15px}` 全站真正放大;
- 密度系统有了作用轴(`data-density` 调 html 字号即可);
- 用户浏览器缩放/OS 字号偏好被尊重(无障碍)。
同步把 15 种散乱字号(10/10.5/11/11.5/12/12.5/13/13.5/14/15/16/17/24…)收敛到 `--fs-*` 七级阶梯(xs 11 / sm 12 / md 13 / base 14 / lg 16 / xl 20 / 2xl 24),半像素字号(10.5/11.5/12.5/13.5)全部归位到相邻档。**290 处改动,纯机械,e2e 静态断言防回归**。

### 1.2 令牌接线与清障(P0)
- 修 2 个未定义引用:`--fg`→`--ink`、`--text`→`--ink`(6 处,工作流编辑器族)。
- 语义色 `-bg/-fg` 全家上岗:所有 `color-mix(in srgb, var(--ok) 12%, transparent)` 手写混色(全站 40+ 处)替换为 `var(--ok-bg)`/`var(--ok-fg)` 引用——一处调整全局生效,这正是当初定义它们的目的。
- `--elev-1/2/3` 接线:卡片/弹层/模态三层投影统一走 token(现在 `--shadow` 一个值打天下)。
- 动效统一 `--dur-fast/base/slow` + `--ease-out`,删除旧 `--dur-1/2`/`--ease` 或做别名。
- 圆角双轨合一:`--radius(10)/--radius-sm(7)` 与 `--r-sm(6)/--r-md(10)/--r-lg(14)` 并存 → 统一到 `--r-*` 四级,旧名做别名过渡。
- 审计徽标两个硬编码色并入引擎色族或语义色;工作流区防御字面量(`var(--x,#hex)`)全部去尾。
- 删除确认无主的死令牌(`--gold-soft`、`--link` 若确认无 JS 引用)。
- **加一件 e2e**:静态扫描 styles.css+app.js,断言 ①无引用未定义 token ②无 #hex 色字面量白名单外新增 ③无 px 字号回潮——把 token 纪律变成回归项(本仓库已有同型静态测试先例 agent-workflow-monitor-ui)。

### 1.3 点击区与密度真联动(P0.5)
`--tap-min` 挂到所有可点原语:`button`(min-height)、`.icon-btn`(32→40 comfortable)、`.session-item`、`.tool-tabs button`、`.skill-toggle`、开关类。`--density-scale` 要么接线(行高/间距乘数)要么删除——不留纸面 token。

## 2. 逐界面设计方向

### 2.1 全局壳层(三栏)
现状:`288px | minmax(420px,1fr) | 340px`,collapse 有显式落轨修复,健壮。
方向:
- **右栏可变宽**:340px 升级为 `340px | 480px | 全屏` 三档(拖拽手柄 + 记忆),Agent 工作流/用量页签激活时默认建议 480px——解决 C 类错位的最低成本方案(§2.7 另有中期方案)。
- 侧栏 288px 保持;新增 `Ctrl+B` 折叠快捷键(现在只有鼠标)。

### 2.2 顶栏
现状:chip 化(工作台/安全/引擎)+ ⋯ 菜单收敛,方向正确;移动端断裂(D)。
方向:
- **P0 修断裂**:`topbar { height:auto; min-height:54px }` 允许两行,或(更优)≤760px 时 chips 降级为图标态(📁→只留图标+首字,perm 只留盾牌,model 只留圆点+短名),外加 `overflow-x:auto` 兜底——顶栏永不裁切。
- 三 chip 视觉统一:当前 workspace(pill)/perm(pill)/model(pill)边框、高度、字号已接近但 padding 各写各的 → 抽 `.chip` 原语类。
- 上下文电量图标保留(好设计),但 `≥60% 才显示` 的简单模式规则改为"活动回合期间常显"——用户反馈"变慢"时第一个要看的就是它。

### 2.3 侧栏(会话列表)
现状:分组标签 + hover 露操作,中规中矩。
方向:
- 会话项操作(置顶/改名/删)hover-only → **触屏不可达**;改为"⋯"常驻小按钮(简单模式)+hover 增强(专家)。
- 会话项增加引擎小圆点(青花蓝/釉里红)——多引擎混用时一眼识别会话归属,数据已有(session.engine)。
- 空搜索结果给空态文案(现在是白板)。
- 「⚙ 设置 / 🩺 体检 / ?」三按钮:体检简单模式已藏,剩两个;把「工作台记忆」入口提到这里(现在只藏在命令面板里,主画像永远发现不了)。

### 2.4 首页/空态(任务卡)
现状:品牌标 + 引擎状态 pill + playbook 卡网格,首跑引导卡完整;云纹水印 opacity .04 实际不可见。
方向:
- 云纹水印 .04→.07(仍克制但真的能看见),或改为 hero 区右上角一枚 10% 摆件——品牌存在感不能靠显微镜。
- 任务卡增加**分组/滚动策略**:现在 10 张卡一屏平铺,技能/Playbook 生态起来后会爆;设计 8 张上限 + 「查看全部」进技能库。
- 任务卡 hover `translateY(-1px)` 太羞涩,配 `--elev-1→2` 投影变化。
- 引导卡的引擎状态区做成**可点击直达设置对应页签**(现在 warn 态有按钮,ready 态不可点——ready 态也应可点进引擎管理)。

### 2.5 对话流(消息/工具卡/计划卡/子代理卡)
现状:880px 阅读宽、气泡+侧头像、工具卡状态条(呼吸动画)、计划卡决策折叠、子代理缩进卡、同回合工具折叠组——**这是全应用完成度最高的界面**,方向只做精修:
- 头像:字母方块 → 用户=中性人形剪影、助手=如意云头小标(SVG,与品牌一致);系统消息头像弱化为无底色图标。
- 工具卡在简单模式显示人话动词(已有),但**参数摘要仍是 mono 原文**(`{"path":"C:\\..."}`);简单模式给参数也做人话化摘要(「写入 报告.md」),原文进详情。
- `msg-actions` hover-only → 简单模式常显(轻透明度),同 2.3 理由。
- 错误卡(error-card)已人性化;补一类**网络断连的行内重试卡**(现在断流只有 toast)。
- 长会话「加载更早」pill 已有;补**日期分隔线**(今天/昨天),长会话定位成本高。

### 2.6 Composer
现状:focus 光环、拖放提示、附件 pill、≤560 折叠进 ＋,结构好。
方向:
- **简单模式技能入口失踪案(P0.5)**:`skillBtn` 简单模式被藏,唯一入口是输入 `/`——主画像不知道 `/`。改为简单模式显示「✨ 技能」按钮(文案从 ⌘ 改掉,⌘ 是 Mac 心智,Windows 产品用它很怪)。
- 发送按钮运行中变「⏹ 停止」(现在停止靠 Esc/顶栏,主画像找不到)。
- 输入框字数/token 轻提示(右下 hint 已有位),接近上下文告警时变琥珀。

### 2.7 右侧工具面板(v1.5 信息重心所在,本稿最重要的结构方向)
现状:6 常驻 + 5 开发者页签;pro 模式 11 页签在 340px 里换 2 行(76px);工作流监控/任务池审批/用量看板全挤 312px 内容宽。
方向(分三步走):
1. **P0.5 页签减负**:简单模式常驻从 6 收到 4(文件/产物/变更 + 「AI 工作」聚合页签——把 Agent 工作流+用量并成一个带子切换的页签);审计挪进 ⋯ 菜单(简单模式)。专家模式页签两行可接受,但组标「开发者」改为图标分隔以省一行。
2. **P1 宽度三档**(§2.1):监控/用量页签激活时自动扩到 480px。
3. **P2 「工作台」视图**:Agent 工作流监控升格为**中栏可切换的全宽视图**(对话 ↔ 工作台 Tab)——DAG 画布式监控(节点卡平铺→图布局)、任务池审批侧板、用量迷你仪表盘同屏。这是团队模式作为核心卖点应得的舞台,也与 UI-ORCHESTRATION-REDESIGN §2 的画布愿景衔接。

### 2.8 用量看板
现状:段控范围切换 + 统计瓦片 + 手绘 SVG 条形/趋势 + 预算横幅 + 诚实脚注,信息设计好;窄。
方向:
- 312px 下三列统计瓦片(17px 数字)拥挤 → 两列;宽面板(480px+)恢复三列(container query 或 JS 类)。
- 趋势图 56px 高无坐标提示 → hover tooltip(纯 CSS title 即可起步)。
- 「计划内计费」徽标是关键诚实设计,但主画像不懂术语 → tooltip 一句话:「订阅套餐,按月付费,token 不另计钱」。
- 币种混排时(CNY+USD)分行显示已有;给每币种行加淡色币种徽标提高扫读性。

### 2.9 Agent 工作流监控 + 任务池
现状:聚合头/状态徽标/引擎徽标/停滞横幅/插话/审批卡三行人话——语义完备;视觉上卡中卡中卡(run>node>pool)边框嵌套过深,312px 里每层再吃 padding。
方向:
- **嵌套降噪**:node 卡去外边框改分隔线;状态徽标从 pill 缩为色点+文字;层级靠缩进与字重表达,不靠三层边框。
- 运行中节点的当前活动行(v1.4.6 的「生成中 · N 字」)提升到 run 聚合头——收起状态也能看到"现在谁在干嘛"。
- 任务池审批卡:琥珀底正确;「同意添加/不用了」按钮加 `--tap-min`;宽限窗倒计时用细进度条替代纯文字秒数。
- 引擎徽标色(`--wf-claude`=青花/`--wf-provider`=赭)与消息区 `--eng-*` 体系并存 → 统一为一套引擎色映射表(现在同一 Claude 在消息区是 #d97757、在工作流区是青花蓝,**自相矛盾**,P0.5 修)。

### 2.10 工作流编辑器(1160px 模态)
现状:网格画布+节点拖拽+连线+inspector,功能完整;自成一派的 `.workflow-btn` pill 按钮、`--text` 未定义 token 重灾区。
方向:
- token 修复(§1.2)后按钮回归全局原语;
- 节点卡(210px)增加引擎徽标与角色色条,与监控视图视觉同源(现在编辑态和运行态长得像两个产品);
- 画布增加小地图 or 「适应视图」按钮(1400×1000 画布在 780px 视口里容易迷路);
- 空画布给引导浮层(「从模板开始 / 添加第一个节点」)。

### 2.11 设置
现状:7 页签,简单模式收 3,分组合理;第三方端点/计费的文案已很用心。
方向:
- 「服务商」卡片的测试按钮结果只有一行字 → 成功/失败给语义色 pill + 延迟毫秒数;
- Claude CLI 页签信息量大,把「第三方端点(Coding Plan)」独立成第 8 页签(它面向的是完全不同的用户决策);简单模式不受影响;
- 每页签顶部一句"这页管什么"的 muted 引导(现在只有 providers 有)。

### 2.12 技能库 / 工作台记忆
现状:640px 模态、35 项单列平铺、内置技能英文名+英文描述、启用 toggle。
方向:
- **内容中文化(P0.5,数据层)**:18 个内置 SKILL.md 补 frontmatter `name-zh`/`description-zh`(或直接改 name/description 为中文,英文 id 保留为目录名)——前端无改动,注册表天然透出。这是主画像可用性问题,不是美化。
- 布局:单列列表 → 两列卡片网格(модal 640px 装得下),启用态卡片青花描边;
- 分组导航:三组(技能/命令/一键任务)加锚点或子页签,35+ 项不能靠滚;
- 记忆面板同构改造;「存为记忆」按钮在 Claude 引擎会话隐藏(现在只在 provider 显示起草按钮,但面板入口本身要保:手写新建两引擎都可用)。

### 2.13 弹层体系(popover/modal/palette/toast)
现状:popover 原语统一、modal 进场动画、palette 标准、toast aria-live——健康。
方向:
- modal 尺寸阶梯:small(420)/default(640)/settings(720)/workflow(1160) → 收敛为 sm/md/lg/xl token;
- toast 增加「操作型」变体(带一个动作按钮,如「已插话 · 查看节点」直达运行卡)——现在动作结果与去处脱节;
- 确认弹层(confirm-body/foot)与 settings modal 是两套 DOM 结构 → 抽一个 `openConfirm()` 原语(app.js 里已散落 3 处相似实现)。

### 2.14 移动端(≤760px)
现状:抽屉化正确;顶栏裁切(D)、侧栏开机盖脸、hover-only 操作不可达。
方向(P0 一揽子):
- 顶栏两行自适应(§2.2);
- ≤760px 开机默认收起侧栏(JS 一行),☰ 唤出;
- hover-only 的 msg-actions/session-actions/ftree-at 在触屏(`(hover:none)` 媒体查询)常显;
- 发送按钮 72×36 → min-height 44(iOS 拇指标准)。

### 2.15 图标语言(横切,P1)
emoji → **内联 SVG 线性图标集**(16/20 两档,stroke 1.5,零依赖、构建期内联)。范围:顶栏(📁🛡🧰⋯)、composer(📎✨)、侧栏(⚙?)、页签、审计源徽标、工具卡 tc-icon。风格基准:如意云头曲线的圆角矩线,青花蓝单色,hover 实心。emoji 保留区:playbook 卡片 icon(用户数据,天然 emoji)与 toast 情绪符。约 24 枚,一次画完进 `icons.svg` sprite 或 JS 常量模块。

## 3. 分级 UI 战略 v2

v1 的分级=「专家全量,简单做减法(藏)」。v2 升级心智:**简单模式不是删减版,是任务优先版**。
- 简单:任务卡是首页、右栏 4 页签、参数人话化、技能按钮显性、常显操作按钮、rem 字号放大真生效;
- 专家:紧凑密度、11 页签、mono 原文、快捷键全量——不动;
- 新增第三态「**演示密度**」(可选,`data-density="presentation"`,html 17px):投屏/教学场景,设置里一键。成本≈0(rem 改造的副产品)。
