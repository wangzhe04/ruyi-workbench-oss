# UI 设计 r2 说明:P2/P3 美学跃升(相对已批准的 r1)

状态:设计稿 r2(2026-07-10)。r1 已批准,本稿只动视觉与质感层,信息架构(四区布局 / 前后对比结构)不变。
配套:`docs/mockups/p3-workbench-r2.html` · `docs/mockups/p2-refinements-r2.html`(r1 文件保留未动)。

## 1. r2 改了什么、为什么(按设计杠杆排序)

1. **画布获得「深度台阶」**(新 token `--canvas-bg`)。r1 画布与面板同底色,节点卡的投影没有物理解释、浮不起来。r2 画布比 `--panel` 低半档(墨夜更沉 `#0c1119` / 月白更清 `#edf1f8`),卡片一放上去就"浮"了——全稿最便宜、收益最大的一笔。
2. **状态从「色块」改为「渗透」**。r1 节点靠 3px 实色左边框 + 淡底色块标状态,读起来像贴标签。r2 改为:状态色以 13% 混入卡底做 115° 渐变洗(从状态缘向卡身衰减)+ 2px 圆头发丝缘 + 运行态光晕(新 token `--glow-accent`)+ 排队态虚线缘(「尚未成形」)。状态是事情的温度,应从边缘渗进来。
3. **连线的图形语言精修**。贝塞尔控制点定为端点纵向 ±38(张力约 0.53,呼应云头曲率);每条边加端口点(源=状态色实点、靶=空心环,方向不靠箭头也可读);着色改 SVG `userSpaceOnUse` 渐变——颜色沿依赖方向流动衰减,与节点的「渗透」同一语言;运行边加沿路径粒子(SMIL,`prefers-reduced-motion` 下 CSS 隐藏)。
4. **数字的仪表盘质感**。所有计量文本(tok/成本/时长/迭代/置信度/倒计时)统一 `.num` 工具类(`tabular-nums`),2s 轮询刷新时数字不再横跳;底部用量条重排为「大数字(`--fs-xl`)+ 微标签」三列仪表 + 发丝竖分隔。顺手修正:r1 mockup 的 `--fs-xl:24px` 与 app 实际阶梯(xl=20 / 2xl=24)不一致,r2 已对齐。
5. **鎏金从「装饰」收敛为「授权清单」**。金子只在完成/通过时刻发光才贵。r2 全稿鎏金仅两处:已完成 run chip 的 ✦ 微星、置信度环弧端的金针尖;形态只许发丝线/微标(新 token `--gold-line`),禁止大面积填充。
6. **云纹从水印升格为结构纹理**。r1 是居中大水印(还是要靠显微镜)。r2 给三个结构位:画布右下角锚饰(裁切云头,opacity .05)、用量条 repeat-x 锦缎暗纹(opacity .045)、run 选择器标签前微云符——全部走既有 `--ruyi-cloud` mask token 上色 `--brand-qh`,随主题走,零新资源。
7. **空间韵律与字阶**。泳道层标注上移至层上缘(micro 字号 + .06em 字距 + `--panel-veil` 磨砂底),不再与节点卡抢同一行;泳道 V_GAP 64→72(层间呼吸感);右板段头从 13px/700 降为 12px/600 + 字距(节奏:段头轻、内容重);画布工具从三颗散点方钮合并为一枚磨砂胶囊组 + 缩放读数。
8. **P2 监控卡:列表长出「树脊」**。状态点串在一条渐隐纵向脊线上,依赖节点经圆角肘线分支——DAG 结构在列表态也可读,而不是只靠缩进暗示;池卡与 P3 同源(琥珀渗透洗 + 左发丝缘 + 宽限窗 3px 发丝倒计时)。技能库:锚点升级为分段控件,分组题改字距标题 + 渐隐线 + 云纹端符,启用态 = 发丝青花描边 + 顶部青花洗 + 实心图标块——与节点卡同一「渗透」语言,全应用一种口音。

## 2. 新 token(4 个,均两主题分别定值,已在两张 r2 设计图头部落定)

| token | dark(墨夜) | light(月白) | 用途与理由 |
|---|---|---|---|
| `--canvas-bg` | `#0c1119` | `#edf1f8` | 画布底,比 `--panel` 低半档的深度台阶;复用 `--bg` 会让画布与应用壳同层,卡片浮不起来 |
| `--panel-veil` | `color-mix(panel 78%, transparent)` | `color-mix(panel 82%, transparent)` | 画布浮件(工具胶囊/层标注)的半透明磨砂底,配 backdrop-blur;取代散落的手写 color-mix |
| `--glow-accent` | 1px 环 32% + 30px 晕 38% | 22% / 25%(月白更克制) | 运行态的「光渗」,取代通用投影——状态用光表达,不用色块 |
| `--gold-line` | `color-mix(gold 55%, transparent)` | `color-mix(gold 60%, transparent)` | 鎏金唯一授权形态:发丝线/微标描边;把「克制」写进 token 名 |

## 3. 可直并 vs 需评审

**纯视觉、可直接并入 P2/P3 实施**:`.num` 工具类;节点渗透渐变 + 发丝缘 + 排队虚线缘;边渐变 + 端口点;tab「笔锋」渐隐下划线;用量条仪表化;画布工具胶囊组;技能卡/分段导航/分组题全套;鎏金与云纹规则;`--canvas-bg`/`--panel-veil`/`--gold-line`/`--glow-accent` 四 token。

**需评审的改动**(动了结构或实现常量):
- 泳道 V_GAP 64→72、画布 720×648:动 `layoutDAG` 常量(P3 稿 §5.2),需同步伪码与 e2e 断言坐标;
- 监控列表树脊/肘线:节点行 DOM 外加一层定位容器,属 P2 实施小改,建议采纳;
- 运行边 SMIL 粒子:锦上添花件。实施建议降级为已有的 CSS dash-flow(r1 方案)或 CSS `offset-path`;SMIL 与 reduced-motion 的协同需要额外守护,砍掉不伤骨架;
- 画布级运行区光晕:mockup 为静态定位,实装需随 running 节点移动——建议只保留节点自带 `--glow-accent`,画布晕作可选增强。

## 4. r1 vs r2 择优建议

建议以 **r2 为合并基线**:信息架构与 r1 完全一致,实现面几乎不增(渐变/光晕/掩模全是 CSS,连线渐变是 SVG defs 数行),但质感档位完全不同。若排期紧,最高性价比五件套 = `--canvas-bg` 深度台阶 + 渗透式节点卡 + 渐变边/端口点 + `.num` 仪表数字 + 鎏金授权清单;SMIL 粒子与画布晕可无痛舍弃。r1 仅在需要最保守 diff 时作为回退基线保留。
