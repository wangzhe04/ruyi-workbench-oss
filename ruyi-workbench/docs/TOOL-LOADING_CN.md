# 工具提示词按需装载（v1.6.1）

Ruyi 默认使用 `toolLoadingMode: "auto"`，同时覆盖 OpenAI 兼容引擎与 Claude CLI。高级设置中的“全部常驻（兼容模式）”可恢复旧行为。

## 三阶段

1. **任务预路由**：每个回合先按用户消息与附件判断文件读取、文件修改、代码、终端、联网、桌面、Office、归档、Agent、Skills 等工具包。纯聊天只保留规划、提问和工具发现入口。
2. **运行中发现与装载**：OpenAI 兼容引擎通过 `tool_search` 查询紧凑目录、通过 `tool_load` 在下一次模型调用加入具体 schema。Claude CLI 每回合启动时接收预路由工具；遗漏能力通过 `tool_search` 与 `tool_invoke_read/edit/exec` 分级代理调用，不依赖 CLI 是否支持运行中刷新 `tools/list`。
3. **缓存、计量与兼容**：外部 MCP 目录按配置缓存（默认 60 秒）；实际注入的 schema token 计入上下文估算与自动压缩判断；事件流报告初始/可用工具数、工具包与 schema token。子代理仍使用自身角色、tier 和白名单，不继承父回合动态装载状态。

## 权限边界

- 动态代理按 `read`、`edit`、`exec` 拆分，调用前再次计算真实目标 tier；用低 tier 代理调用高 tier 工具会被拒绝。
- 外部 Office 写操作仍要求绝对路径并进入检查点日志；原有权限模式、工作区护栏与桥接工具 tier 覆盖继续生效。
- Claude exec-tier 工作流节点保留原有直接 MCP 白名单合同；主聊天回合使用紧凑代理面。
