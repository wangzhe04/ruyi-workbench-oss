# 文档处理

适用于整理、改写、校对、对比或批量处理文档。目标不是“生成一段文字”，而是交付可核验、可继续编辑的文件。

## 工作流程

1. 先列出输入文件，确认格式、数量、大小和目标输出路径；原文件默认只读。
2. 对文本型文件优先直接读取；扫描 PDF 或图片只有在普通提取失败后才走 OCR，并标出 OCR 可能误识别的位置。
3. 改写前先保留标题层级、表格、编号、引用和专有名词；没有明确要求时不改变事实与立场。
4. 多文档对比按“主题或章节 → 文件 A → 文件 B → 差异 → 影响”组织，不只做逐字差异。
5. 输出到新文件，避免覆盖来源；必须覆盖时先说明目标并请求确认。
6. 交付前重新读取结果，检查文件存在、非空、标题完整、没有截断和乱码。

## 输出要求

- 给出来源文件、结果文件和处理范围。
- 区分原文事实、推断、改写和无法确认的内容。
- 列出跳过或读取失败的文件及原因。
- 中文文档优先使用 UTF-8；Office/PDF 产物要检查中文字体显示。

## 停止条件

输入缺失、文件受密码保护、格式无法解析，或要求会覆盖大量原文件时，停止并说明可执行的下一步，不伪造处理结果。
