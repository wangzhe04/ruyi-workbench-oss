# 结构化写作

适用于周报、汇报、方案、说明、总结和邮件等成稿任务。先忠实整理事实，再改善结构与表达，不补造不存在的信息。

## 工作流程

1. 明确读者、用途、语气、篇幅、截止时间和输出格式；缺省时采用简洁、专业、可扫描的风格。
2. 从素材中提取事实、数字、决定、行动项、风险和待确认项，并保留原始措辞中的关键限定。
3. 先搭建“结论/摘要 → 依据 → 细节 → 下一步”的结构，再写正文。
4. 删除重复、空话和无法支撑的强结论；统一日期、数字、术语与人名。
5. 对未知信息使用明确占位符，例如 `[待补：负责人]`，绝不虚构。
6. 输出后做一次编辑检查：标题层级、逻辑跳跃、指代、错别字、行动项是否包含负责人和时间。

## 常用交付结构

- 汇报：关键进展、结果/指标、问题风险、下周计划、需协助事项。
- 方案：背景、目标与非目标、选项比较、建议方案、实施步骤、风险与验收。
- 会议纪要：会议结论、讨论要点、行动项（负责人/期限）、待定问题。

## 输出要求

给出可直接使用的成稿；若素材存在冲突或缺口，在正文之外单列“待确认”，不要把疑点悄悄写成定论。
